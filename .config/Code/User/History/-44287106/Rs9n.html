Vim Way To Deal With Repeation

  • Our work is repetitive by nature.
  • Vim is optimized for repetition.
  • We can always replay the last change with a single keystroke.
It’s useless unless we learn to craft our actions so that they perform a useful unit of work when replayed. Mastering this concept is the key to becoming effective with Vim.

  • But what is a Change?
  • A change could be an action:
    1. on individual characters.
      2. > do the exercises in file: 1-repeation.md The x command deletes the character under the cursor. When we use the dot command in this context, “repeat last change” tells Vim to delete the character under the cursor
    2. on entire lines.
      3. > do the exercises in file: 2-repeation.md The dd command also performs a deletion, but this one acts on the current line as a whole. If we use the dot command after dd, then “repeat last change” instructs Vim to delete the current line
    3. or even the whole file.
      4. > do the exercises in file: 3-repeation.md Finally, the >G command increases the indentation from the current line until the end of the file. If we follow this command with the dot command, then “repeat last change” tells Vim to increase the indentation level from the current position to the end of the file.
